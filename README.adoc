= CLI url checker tool

A command-line tool for checking broken links on web pages.

== Usage

----
Usage: url_checker [options]

Checks for broken links on the pages provided (urls)

Options:
  -s, --selector [css selector]  CSS selector for only checking part of a page (such as: "main" for the <main> element).
  -o, --output [file]            File to output the results (default: "url_checker_results.csv")
  -f, --format [format]          File format to output the results (supports: "csv", "json", "yaml", "sarif") (default: "csv")
  -u, --url <url>                URL of the page to check including protocol (such as https://www.example.com/index.html)
  -h, --help                     display help for command
----

== Output Formats

=== CSV

Flat comma-separated values format with all URL check results.

[source,bash]
----
url_checker -u https://example.com -f csv -o results.csv
----

=== JSON

Structured JSON format with results grouped by parent URL.

[source,bash]
----
url_checker -u https://example.com -f json -o results.json
----

=== YAML

Human-readable YAML format with results grouped by parent URL.

[source,bash]
----
url_checker -u https://example.com -f yaml -o results.yaml
----

=== SARIF

https://docs.oasis-open.org/sarif/sarif/v2.1.0/sarif-v2.1.0.html[SARIF v2.1.0] format for integration with CI/CD pipelines and code scanning tools. Includes:

* Line and column numbers for each URL found in the source
* Rule definitions for different types of URL issues
* Severity levels (error, warning, note)
* Full artifact and invocation metadata

[source,bash]
----
url_checker -u https://example.com -f sarif -o results.sarif
----

== Features

* Checks HTTP/HTTPS links for broken URLs (404, 500, etc.)
* Validates anchor links within pages
* Tracks duplicate URLs on different lines
* Supports CSS selectors to check specific page sections
* Multiple output formats for different use cases
* Line-accurate reporting in SARIF format
